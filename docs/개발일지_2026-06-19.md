### 1. 구역장 수동 출석 편집 (`ScheduleManagement.tsx`)

**배경:** 구역모임 이후 구역원이 직접 출석 체크를 안 하면 데이터 공백 발생.
구역장이 모임 후 전체 구역원 출석을 수동으로 입력할 수 있어야 함.

**구현:**
- `AttendanceStatus` 컴포넌트에 `districtId` prop 추가
- 구역장 뷰: `getAllUsers(districtId)`로 전체 active 구역원 목록 로드
- 각 구역원 행 클릭 → 참석 ↔ 불참 즉시 토글 (미응답은 회색 아이콘)
- 구역원 뷰: 기존 "참석/불참" 버튼 그대로 유지
- 구역장은 과거 일정(`isPast`)에서도 편집 가능 — 모임 후 수동 입력 시나리오 대응

---

### 2. Dashboard 섹션 레이블 + "우리 구역은?" strip (`Dashboard.tsx`)

**배경:** 구역장 대시보드에 카드가 늘어나면서 "관리해야 할 것" vs "내가 해야 할 것"이 섞여 보임.
지난주 구역 현황도 구역원과 공유하면 소속감 강화.

**구현:**
- 구역장에게만 `구역 운영` / `나의 활동` 섹션 레이블 추가 (tracking-widest uppercase, 아주 작게)
- "우리 구역은?" strip: 가장 최근 locked 주간보고 기준 성경읽기 총 장수 + 출석 인원 표시
  - 데이터 소스: `getWeeklyReports(districtId)` → `isLocked === true` 최신 항목
  - 마감된 보고서가 없으면 미표시

**구역원도 볼 수 있음** — 지난주 우리 구역이 함께 한 활동을 자연스럽게 공유

### 3. 구역 선택 목록에 교회명 표시 (`DistrictPicker.tsx` + 마이그레이션 049)

**배경:** 멀티테넌트 환경에서 회원가입 시 구역 목록에 교회명이 없어 어느 교회 구역인지 알 수 없음.
`anon` 역할은 `churches` 테이블 직접 접근 불가 (034 RLS 제약).

**구현:**
- 마이그레이션 `049_districts_with_church_fn.sql`: `get_active_districts_with_church()` RPC
  - `SECURITY DEFINER` — anon/authenticated 모두 호출 가능
  - `districts JOIN churches` 결과 반환 (deleted_at 필터 포함)
- `getActiveDistricts()`: 기존 직접 쿼리 → RPC 호출로 교체, `churchName` 필드 추가
- `DistrictPicker`: 교회가 2개 이상일 때 `킨텍스장성남 (벧엘교회)` 형식으로 표시
  - 교회가 1개(단일 테넌트 상황)면 교회명 생략

**적용 대상:** 회원가입 구역 선택 (`Login.tsx`), 구역 변경 (`PendingApproval.tsx`)
