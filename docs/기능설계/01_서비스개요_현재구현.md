# 서비스 개요 및 현재 구현

## 1. 프로젝트 개요

**프로젝트명:** 벧엘교회 킨텍스장성남 구역 관리 시스템  
**기술 스택:** React 18 + TypeScript + Vite + Tailwind CSS + shadcn-ui + Supabase + Firebase Hosting  
**현재 운영 형태:** React SPA + Supabase(Auth, PostgreSQL, Storage, Edge Functions) 기반 운영 구조

### 1.1 목적

교회 구역 단위의 활동을 디지털로 관리한다. 주요 관리 대상은 성경공부, 기도제목, 성경읽기, 일정, 출석, 주간 운영 보고다.

### 1.2 사용자 유형

| 유형 | 식별 방법 | 권한 |
|------|----------|------|
| 구역원 | `users.role = member` | 개인 활동 입력, 일정 조회, 출석 응답 |
| 구역장 | `users.role = leader` | 구역원 권한 + 관리자 기능 |
| 마스터 | `users.role = master` | 전체 구역 관리, 구역 생성/전환 |

### 1.3 현재 시스템 한계

- 기능 구현은 많이 진행됐지만, 운영 환경에서 `cron`, Edge Function, Storage 설정이 실제 활성화됐는지는 별도 확인이 필요하다.
- 일부 설계 문서는 과거 `localStorage` 기반 설명을 아직 포함하고 있었다.
- PWA/오프라인은 패키지 의존성은 있으나 실제 활성화 상태로 보기 어렵다.
- 관리자 화면 번들이 커서 운영 성능 최적화 여지가 있다.

## 3. 현 단계 판단

현재 앱은 더 이상 "단일 브라우저 시범판" 단계가 아니다. 코드 기준으로는 핵심 업무 기능과 운영 기능이 대부분 Supabase 기반으로 전환된 상태다.

현재 단계는 다음과 같이 보는 것이 적절하다.

1. 핵심 기능 개발: 대부분 완료
2. 운영 자동화 코드: 구현 완료에 가까움
3. 운영 환경 활성화: 최종 점검 필요
4. 문서 및 성능 정리: 후속 작업 필요

즉, 지금 남은 핵심 과제는 신규 기능 대량 추가보다 운영 준비와 안정화다.

## 4. 현재 우선 과제

1. Supabase 운영 환경 변수, Edge Function, Storage, `pg_cron` 실제 적용 여부 확인
2. 역할별 실사용 점검
3. 문서 최신화
4. 관리자 화면 성능 최적화
5. PWA 포함 여부 결정
