# 서비스 개요 및 현재 구현

## 1. 프로젝트 개요

**프로젝트명:** 벧엘교회 킨텍스장성남 구역 관리 시스템  
**기술 스택:** React 18 + TypeScript + Vite + Tailwind CSS + shadcn-ui + localStorage  
**현재 운영 형태:** 프론트엔드 SPA, 서버 없음, 정적 배포

### 1.1 목적

교회 구역 단위의 활동을 디지털로 관리한다. 주요 관리 대상은 성경 공부, 기도 제목, 성경 읽기, 일정, 출석이다.

### 1.2 사용자 유형

| 유형 | 식별 방법 | 권한 |
|------|----------|------|
| 구역원 | 일반 사용자 | 개인 활동 입력, 일정 조회, 출석 응답 |
| 구역장 | 이름에 "구역장" 포함 | 구역원 권한 + 일정/관리자 기능 |

### 1.3 현재 시스템 한계

- 인증이 로컬 검증 기반이라 실제 사용자 식별이 불완전하다.
- 모든 데이터가 `localStorage`에 저장되어 기기 간 동기화가 되지 않는다.
- 관리자 일부 탭은 UI만 존재하고 실제 데이터 기능이 없다.

## 2. 현재 구현 범위 요약

| 영역 | 현재 상태 | 핵심 비고 |
|------|----------|----------|
| 인증 | 기본 로그인/로그아웃 구현 | 회원가입 요청, 비밀번호 재설정은 UI만 |
| 대시보드 | 통계 카드, QT 링크, 최근 공부, 일정 팝업 구현 | 실데이터 집계는 로컬 기준 |
| 성경 공부 | 목록/상세/답변 저장 구현 | 자료 3건 하드코딩 |
| 기도 제목 | 등록, 목록, 응답 처리 구현 | 전체가 로컬 저장 |
| 성경 읽기 | 일일 기록, 누적 진도 시각화 구현 | 연간 누적 계산 포함 |
| 일정 관리 | 일정 CRUD, 출석 응답 구현 | 첨부는 URL만 지원 |
| 관리자 | 개요/기도/일정 일부 실동작 | 접속 로그, 공부 관리, 내보내기 일부 미구현 |

## 3. 현 단계 판단

현재 앱은 "단일 브라우저에서 운영 가능한 시범판" 수준이다. 실제 운영을 위해서는 다음 3가지가 선행되어야 한다.

1. 서버 기반 인증과 사용자 승인 체계
2. DB 기반 영속 저장과 권한 제어
3. 관리자 운영 기능의 실데이터 전환
