# 개발 로드맵

## 1. 우선순위 원칙

- 백엔드 없이 가능한 개선을 먼저 처리한다.
- 다음 단계의 전제 조건을 선행한다.
- 구역원이 체감하는 기능과 운영 리스크 감소를 우선한다.
- 자동화는 수동 기능 완성 후 붙인다.

## 2. 우선순위 요약

| 우선순위 | 항목 |
|---------|------|
| P1 | 답안 프라이버시, 카카오 공지, 백엔드 구축 |
| P2 | 묵상 자동화, 주간 마감, 주간 보고서 |
| P3 | 주보 PDF 자동 수집, 부가 기능 고도화 |

## 3. Phase별 진행 순서

### Phase 1. 프론트엔드 즉시 개선

| 순서 | 작업 | 비고 |
|------|------|------|
| 1-1 | CSS `@import` 순서 오류 수정 | 빌드 경고 제거 |

| 1-4 | `StudyAnswer`에 `userId`, `userName` 추가 | 1-3 전제 |
| 1-5 | 카카오톡 공지 메시지 생성 UI | 실사용 즉시 가능 |
| 1-6 | 다크 모드 토글 | 기존 설정 활용 |

## 5. 문서 운영

- 일정과 우선순위 변경은 이 문서를 기준으로 반영한다.
