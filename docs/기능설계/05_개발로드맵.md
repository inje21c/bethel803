# 개발 로드맵

## 1. 문서 목적

이 문서는 최초 계획표이면서 동시에 현재 시점의 진행 상태를 반영하는 관리 문서다.

- 원래 설계된 Phase 순서를 유지한다.
- 현재 저장소 코드 기준으로 완료 여부를 다시 표시한다.
- 앞으로 남은 작업은 "신규 개발"보다 "운영 활성화와 안정화" 중심으로 정리한다.

## 2. 현재 종합 판단

현재 코드 기준 진행 상태는 다음과 같다.

| 구분 | 현재 판단 |
|------|----------|
| 핵심 기능 개발 | 대부분 완료 |
| 백엔드 전환 | 완료에 가까움 |
| 운영 자동화 코드 | 구현됨 |
| 운영 환경 활성화 | 확인 필요 |
| 문서 정리 | 진행 중 |
| 성능/PWA | 일부 잔여 |

요약하면, 현재 단계는 대규모 신규 기능 개발보다 운영 준비와 후속 정리가 우선이다.

## 3. 우선순위 원칙

- 이미 구현된 기능은 재개발하지 않고 운영 활성화 여부를 먼저 확인한다.
- 사용자 체감 기능보다 운영 리스크 제거를 우선한다.
- 자동화는 코드 구현과 실제 스케줄 등록을 분리해서 판단한다.
- 문서와 실제 코드 상태 차이를 줄인다.

## 4. 현재 우선순위 요약

| 우선순위 | 항목 |
|---------|------|
| P1 | Supabase 운영 환경 점검, Edge Function 배포, `pg_cron` 등록 확인 |
| P2 | 역할별 실사용 점검, 관리자 기능 검증, 문서 최신화 |
| P3 | 관리자 화면 성능 최적화, PWA 적용 여부 결정 |
| P4 | 설치형 앱 전환, 구독 알림, AI 기능 MVP 기획 및 단계적 적용 |
