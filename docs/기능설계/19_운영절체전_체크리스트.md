# 운영 절체 전 체크리스트

## 1. 문서 목적

이 문서는 `Firebase 운영 -> Vercel 운영` 절체 직전에 반드시 확인해야 할 항목을 정리한다.

현재 기준:

- `staging Supabase` 준비 완료
- `Vercel preview` 연결 완료
- 로그인, 성경공부, 주보 파싱, 원본 등록, 기본 회귀 확인 완료

따라서 이제는 신규 기능 구현보다 `운영 절체 기준 충족`을 확인하는 것이 우선이다.

## 2. 절체 판단 원칙

- `preview 성공`만으로 절체하지 않는다.
- `production 리허설 성공`까지 확인해야 한다.
- `롤백 경로`가 준비되지 않으면 절체하지 않는다.
- `Firebase 운영 경로`는 컷오버 직후에도 바로 제거하지 않는다.

한 줄 원칙:

`staging 검증 -> production 리허설 -> 절체 -> 롤백 대기` 순서로 간다.

## 3. 필수 완료 기준

아래가 모두 충족되면 운영 절체를 진행할 수 있다.

- [ ] Vercel production env 입력 완료
- [ ] Supabase prod Auth Redirect URL 정렬 완료
- [ ] production 빌드/배포 성공
- [ ] production 로그인 성공
- [ ] 역할별 기본 화면 접근 성공
- [ ] 성경공부 조회/수정/발행 성공
- [ ] 첨부 업로드 성공
- [ ] 주보 파싱 핵심 경로 확인 완료
- [ ] Firebase 롤백 경로 유지 확인

## 4. 체크리스트

### 4.1 환경 변수 및 연결

- [ ] Vercel `Production`에 prod `VITE_SUPABASE_URL` 입력
- [ ] Vercel `Production`에 prod `VITE_SUPABASE_ANON_KEY` 입력
- [ ] Vercel `Production`에 prod `VITE_APP_URL` 입력
- [ ] prod Supabase Auth Redirect URL에 production 도메인 추가
- [ ] prod Supabase Edge Function secret 확인

### 4.2 빌드 및 배포

- [ ] `main` 또는 절체용 브랜치에서 production 배포 성공
- [ ] 라우팅(`/dashboard`, `/admin`, `/bible-study/:id`) 정상
- [ ] 새로고침 시 SPA 라우팅 정상

### 4.3 로그인 및 권한

- [ ] `master` 로그인 성공
- [ ] `leader` 로그인 성공
- [ ] `member` 로그인 성공
- [ ] 로그인 후 `/login`으로 튀는 현상 재현되지 않음
- [ ] 로그아웃 후 자동 로그인 재진입 현상 재현되지 않음
- [ ] `pending` 계정은 `/pending`으로 이동
- [ ] `member`는 `/admin` 차단
- [ ] `leader`는 `/admin` 접근 가능
- [ ] `master`는 `/districts` 접근 가능

### 4.4 핵심 기능 회귀

- [ ] 대시보드 진입
- [ ] 성경공부 목록 조회
- [ ] 성경공부 상세/답안 저장
- [ ] 기도제목 목록/상세
- [ ] 성경읽기 기록
- [ ] 일정 등록/조회/출석
- [ ] 주간 보고 조회
- [ ] 관리자 구역원 승인

### 4.5 성경공부 원본/수정본 흐름

- [ ] 주보 파싱 원본 등록
- [ ] 같은 날짜 재파싱 시 `409` 경고 확인
- [ ] 원본 목록 조회
- [ ] `내 구역 수정본 만들기` 성공
- [ ] 수정본 편집 성공
- [ ] 수정본 발행 성공
- [ ] 구역원은 자기 구역 발행본만 조회

### 4.6 업로드 및 외부 연동

- [ ] 첨부 업로드 성공
- [ ] 첨부 URL 열람 성공
- [ ] `parse-bulletin` 함수 호출 성공
- [ ] `fetch-devotional` 운영 경로 영향 없음 확인

### 4.7 운영 안정성

- [ ] 브라우저 새로고침 후 세션 유지
- [ ] 모바일 Safari에서 로그인/성경공부 흐름 정상
- [ ] 주요 에러 콘솔에 치명적 반복 없음
- [ ] Supabase logs에 500/401 반복 없음

### 4.8 롤백 준비

- [ ] Firebase 운영 배포 경로 유지
- [ ] 기존 운영 URL 확인 가능
- [ ] 롤백 시 되돌릴 env 값 목록 준비
- [ ] 절체 직후 문제 발생 시 복귀 절차 공유

## 5. 절체 직전 보류 항목

운영 절체 전에는 아래를 같이 묶지 않는 편이 좋다.

- [ ] `013_phase8_push_notifications.sql` prod 적용
- [ ] 실제 웹푸시 발송
- [ ] AI 추가 기능 prod 반영
- [ ] 대규모 UI 리디자인

즉, 운영 절체는 `기존 핵심 기능 안정화`만으로 끝내는 것이 좋다.

## 6. 절체 후 바로 볼 것

- [ ] 첫 로그인 성공 여부
- [ ] 성경공부 발행본 노출 여부
- [ ] 관리자 화면 접근 여부
- [ ] 첨부 업로드 여부
- [ ] 에러 로그 15분 모니터링

## 7. 최종 판단 문장

아래 문장을 자신 있게 말할 수 있으면 절체 가능 상태다.

`Vercel production은 현재 Firebase 운영과 기능적으로 동등하고, 문제가 생겨도 즉시 롤백할 수 있다.`
