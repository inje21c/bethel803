# 슈퍼어드민 QT 모드 관리 설계

작성일: 2026-06-18

## 배경

- fetch-devotional Edge Function이 매일성경(성서유니온)을 스크래핑해 QT 콘텐츠를 제공
- 초기에는 벧엘교회 church_id를 하드코딩
- SaaS 다교회 환경에서는 슈퍼어드민이 특정 교회에 scraped 모드를 허가해야 함
- 2026-06-18: fetch-devotional 다교회 동적 조회로 변경 완료

## QT 모드 종류

| 모드 | 설명 | 대상 |
|---|---|---|
| `simple` | 자체 성경 구절 (시편 등 1일 1편, 외부 의존 없음) | 신규 교회 기본값 |
| `scraped` | 매일성경 스크래핑 (성서유니온 라이선스 필요) | 슈퍼어드민 허가 교회만 |
| `admin` | 관리자가 직접 등록 | 향후 구현 예정 |

## 운영 플로우

```
교회 셀프가입 → qt_mode = simple (기본)
      ↓
교회가 매일성경 연동 요청 (고객지원 문의)
      ↓
슈퍼어드민 대시보드 → 해당 교회 편집 → QT 방식: scraped 선택 → 저장
      ↓
다음 날 06:00 KST cron 자동 실행
→ church_settings.qt_mode='scraped' 교회 목록 조회
→ qt_contents 생성 (스크래핑 or 기존 데이터 복사)
      ↓
당일 즉시 적용 필요 시: fetch-devotional 수동 호출
→ 기존 scraped 교회 데이터에서 복사 (재스크래핑 없음)
```

## 구현 범위

- `updateChurchSuperAdmin()` 함수에 `qtMode` 파라미터 추가

- EditModal에 QT 방식 드롭다운 추가
- 교회 목록 테이블에 QT 모드 뱃지 표시
