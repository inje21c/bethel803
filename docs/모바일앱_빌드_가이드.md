# 모바일 앱(Capacitor) 빌드 가이드

기존 웹앱(React+Vite)을 네이티브 셸로 감싼 테스트 빌드 절차.
결제(IAP)·커스텀 도메인·사업자등록은 **정식 스토어 공개 때만** 필요. 테스트는 무관.

## 구성
- `capacitor.config.ts` — appId=`kr.or.bethel.guzic`, webDir=`dist`
- `android/` — Android 네이티브 프로젝트 (Android Studio로 빌드)
- `ios/` — iOS 네이티브 프로젝트 (Mac + Xcode 필요)
- npm 스크립트: `cap:sync`, `cap:android`, `cap:ios`

## 사전 준비 (각 PC 1회)
- 공통: Node, `npm install`
- Android: Android Studio + Android SDK (JDK 포함). 실기기 USB 디버깅 또는 에뮬레이터
- iOS: macOS + Xcode + CocoaPods (`sudo gem install cocoapods`)

## A. 빠른 테스트 (번들 없이 실서버 로드) — 가장 빠름
1. `capacitor.config.ts`의 `server.url` 주석 해제 (배포 URL 로드)
2. `npx cap sync`
3. `npx cap open android` → Android Studio에서 Run
- 장점: 즉시 확인, OAuth 리디렉트도 실 https라 수월
- 주의: 네트워크 필요, 정식 빌드 전 반드시 주석 복원

## B. 번들 테스트 (정석)
1. `server.url` 주석 처리 상태 유지
2. Android: `npm run cap:android` (build+sync+open) → Android Studio Run
3. iOS(Mac): `npm run cap:ios` → Xcode에서 서명팀 선택 후 Run

## 코드 수정 후 반영
- 웹 변경 시: `npm run cap:sync` (build + 네이티브로 복사)
- 네이티브 설정만 변경 시: `npx cap sync`

## 테스트 단계 알려진 제약 (정식 공개 전 처리)
- **푸시**: 현재 Web Push(VAPID). 네이티브 푸시는 APNs/FCM 전환 필요(미구현)
- **OAuth**: 카카오/구글이 인앱 WebView에서 차단될 수 있음 → 시스템 브라우저/네이티브 SDK 필요
- **PWA 서비스워커**: Capacitor 내 캐싱 충돌 가능 → 흰 화면 시 네이티브 빌드에서 SW 비활성화 검토
- **딥링크**(초대링크 `/join`): 커스텀 도메인 + AASA/assetlinks.json 필요
- **후원 계좌 노출**: iOS는 비영리 인증 없으면 거절 위험 → iOS 분기 숨김/외부링크

## 내부 배포(스토어 심사 전 베타)
- Android: Play Console 내부 테스트 트랙 (또는 APK 직접 공유)
- iOS: TestFlight
- 개발자 계정만 있으면 됨. 결제/사업자 불필요
