# bethel803 기능 설계서

**프로젝트명:** 벧엘교회 킨텍스장성남 구역 관리 시스템
**버전:** 1.0 (현황 분석 기준)
**작성일:** 2026-03-09
**기술 스택:** React 18 + TypeScript + Vite + Tailwind CSS + shadcn-ui + localStorage

---

## 1. 프로젝트 개요

### 1.1 목적

교회 구역(소그룹) 단위의 신앙 활동을 디지털로 관리하는 웹 애플리케이션.
구역원들의 성경 공부, 기도 제목, 성경 읽기, 일정 및 출석을 통합 관리한다.

### 1.2 사용자 유형

| 유형 | 식별 방법 | 권한 |
|------|----------|------|
| 구역원 (member) | 일반 사용자 | 성경 공부, 기도 제목, 성경 읽기, 일정 조회/출석 |
| 구역장 (leader) | 이름에 "구역장" 포함 | 구역원 권한 + 일정 관리 + 관리자 대시보드 전체 |

### 1.3 시스템 구성 현황

- **프론트엔드:** React SPA (Single Page Application)
- **백엔드:** 없음 (모든 데이터 localStorage 저장)
- **인증:** 이름/비밀번호 단순 검증 (서버 없음)
- **배포:** 정적 호스팅 방식

## 3. 미구현 기능 (개발 필요 항목)

### 3.1 우선순위 분류

| 우선순위 | 기준 |
|---------|------|
| P1 (필수) | 시스템 운영에 반드시 필요 |
| P2 (중요) | 사용성 향상에 크게 기여 |
| P3 (선택) | 부가 편의 기능 |

---

### 3.2 P1 — 필수 개발 항목

#### 3.2.1 백엔드 API 및 데이터베이스 연동

**현황:** 모든 데이터가 localStorage에만 저장됨. 다른 기기/브라우저에서 데이터 공유 불가.

**필요 기능:**
- REST API 또는 BaaS(Supabase/Firebase) 연동
- 사용자별 데이터 서버 영속화
- 멀티 유저 데이터 동기화

**영향 범위:** 전체 모듈

---

#### 3.2.2 실제 인증 시스템

**현황:** 이름+비밀번호 입력 시 로컬 검증만 수행. 어떤 이름/비밀번호도 입력 가능.

**필요 기능:**
- 서버 기반 사용자 등록 및 로그인
- 비밀번호 해시 저장
- JWT 또는 세션 기반 인증 토큰
- 구역장의 회원가입 요청 승인/거절 처리
- 비밀번호 재설정 이메일 발송

**신규 API 엔드포인트:**
```
POST /api/auth/register     - 회원가입 요청
POST /api/auth/login        - 로그인
POST /api/auth/logout       - 로그아웃
POST /api/auth/reset        - 비밀번호 재설정 요청
PATCH /api/auth/approve/:id - 회원가입 요청 승인 (구역장)
```

---

#### 3.2.3 성경 공부 자료 동적 관리

**현황:** 3개 공부 자료가 코드에 하드코딩됨. 관리자 대시보드 공부 관리 탭은 UI만 존재.

**필요 기능:**
- 구역장이 성경 공부 자료 CRUD
- 질문 목록 동적 추가/편집/삭제
- 공부 자료 발행/비발행 상태 관리

**신규 API 엔드포인트:**
```
GET    /api/studies          - 공부 목록 조회
GET    /api/studies/:id      - 공부 상세 조회
POST   /api/studies          - 공부 등록 (구역장)
PATCH  /api/studies/:id      - 공부 수정 (구역장)
DELETE /api/studies/:id      - 공부 삭제 (구역장)
```

### 3.3 P2 — 중요 개발 항목

**필요 기능:**

**변경 대상:** AdminDashboard.tsx (성경 읽기 탭, 일정 관리 탭)

#### 3.3.3 접속 로그 실제 구현

**현황:** 관리자 접속 로그 탭에 mock 데이터만 표시.

**필요 기능:**
- 사용자 로그인/로그아웃 시각 기록

#### 3.3.4 알림 시스템

**현황:** 새 일정, 기도 응답, 공부 등록 시 구역원에게 알림 없음.

**필요 기능:**
- 앱 내 알림 센터 (벨 아이콘 + 읽음/안읽음)
- 새 일정 등록 시 구역원 알림
- 기도 제목 응답 시 해당 구역원 알림
- 새 성경 공부 등록 시 알림

**신규 컴포넌트:** NotificationCenter.tsx
**신규 API 엔드포인트:**
```
GET   /api/notifications        - 내 알림 목록
PATCH /api/notifications/:id    - 알림 읽음 처리
DELETE /api/notifications/:id   - 알림 삭제
```

#### 3.3.5 구역원 관리 페이지

- 구역원 비활성화/삭제
- 역할(구역장/구역원) 변경

```

```

```
/                      - 로그인
/dashboard             - 대시보드 (홈)
/bible-study           - 성경 공부 목록
/bible-study/:id       - 성경 공부 상세 / 답변 작성
/prayer-requests       - 기도 제목
/bible-reading         - 성경 읽기 기록
/schedule              - 일정 관리
/admin                 - 관리자 대시보드 (구역장 전용)
```

```
/admin/members         - 구역원 관리 (P2)
/notifications         - 알림 센터 (P2)
/profile               - 내 프로필 (P3)
```

## 6. 기술 부채 및 개선 사항

| 항목 | 현황 | 개선 방향 |
|------|------|----------|
| 테스트 코드 | placeholder 1개만 존재 | Vitest + React Testing Library로 단위/통합 테스트 작성 |
| 에러 처리 | Error Boundary 없음 | 글로벌 Error Boundary 컴포넌트 추가 |
| 폼 검증 | 일부 미적용 | React Hook Form + Zod 스키마 전체 적용 |
| 역할 감지 방식 | 이름에 "구역장" 포함 여부로 판단 | 서버 기반 role 필드로 전환 |
| 하드코딩 데이터 | 성경 공부 3개 mock 데이터 | DB 기반 동적 데이터로 전환 |
| 접근성 | aria 속성 미적용 | WCAG 기준 aria-label 추가 |

## 7. 개발 로드맵 (권장)

### Phase 1 — 백엔드 기반 구축 (P1)
1. Supabase 또는 Firebase 연동
2. 실제 인증 시스템 구축 (로그인, 회원가입 승인, 비밀번호 재설정)
3. 전체 데이터 모델 DB로 마이그레이션
4. 기존 localStorage 로직 API 호출로 교체

### Phase 2 — 핵심 기능 완성 (P2)
5. 성경 공부 자료 CRUD (관리자)
6. 파일 업로드 (일정 첨부파일)
7. 데이터 내보내기 (CSV/Excel)
8. 접속 로그 실제 구현
9. 구역원 관리 페이지

### Phase 3 — 사용성 향상 (P2~P3)
10. 통계 차트 시각화 (Recharts 활용)
11. 앱 내 알림 시스템
12. 전역 검색 기능
13. 다크 모드 토글

### Phase 4 — 고도화 (P3)
14. 웹 스크래핑 자동화
15. 푸시 알림 (Web Push API)
16. 오프라인 지원 (PWA/Service Worker)
17. 테스트 코드 작성
