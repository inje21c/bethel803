# qa-roles-bethel803

## 1. 목적

이 문서는 `bethel803`의 역할별 사용자 흐름을 Codex가 일관되게 검증하기 위한 실행 문서다.

검증 대상 역할:

- `master`
- `leader`
- `member`
- `pending`

목표는 다음과 같다.

- 역할별 화면 접근 제어를 확인한다.
- 구역 분리와 권한 경계를 확인한다.
- 운영에 중요한 핵심 사용자 흐름을 시나리오로 검증한다.
- 실패 항목을 버그, 문서 수정, 운영 설정 이슈로 구분한다.

## 2. 언제 사용하는가

다음 상황에서 사용한다.

- 운영 시작 전 최종 역할 검증
- Auth 또는 권한 로직 수정 후
- 멀티구역 관련 변경 후
- 관리자 화면 구조 변경 후
- 배포 직전 회귀 점검

## 4. 입력 정보

실행 전 아래 정보를 준비한다.

- 검증 대상 환경
  - 로컬
  - staging
  - production

- 서로 다른 구역 소속 사용자 정보
- 검증 대상 URL
- 최근 권한/인증 관련 변경 내용

## 5. 선행 조건

- 테스트 계정이 실제로 로그인 가능한 상태여야 한다.
- `pending` 계정은 승인 대기 상태여야 한다.
- 적어도 두 개 이상의 구역 데이터가 있으면 구역 분리 검증이 더 정확하다.
- 원격 검증이 불가능한 경우, 로컬 기준으로 가능한 범위만 수행하고 미확인 항목을 분리한다.

## 6. 점검 원칙

- 권한 검증은 단순 화면 노출이 아니라 실제 접근 가능 여부까지 본다.
- 구역 분리 검증은 "보이지 않음"뿐 아니라 "다른 구역 데이터가 섞이지 않음"까지 확인한다.
- 실패 항목은 반드시 분류한다.
  - 코드 버그
  - 운영 설정 문제
  - 테스트 데이터 부족
  - 문서 불일치

## 7. 역할별 기대 동작

### 7.1 master

- 일반 사용자 기능 접근 가능
- 관리자 기능 접근 가능
- `/districts` 접근 가능
- 전체 구역 관리 가능

### 7.2 leader

- 일반 사용자 기능 접근 가능
- `/admin` 접근 가능
- 자기 구역 범위 내 관리 기능 사용 가능
- `/districts`는 접근 불가

### 7.3 member

- 일반 사용자 기능 접근 가능
- `/admin` 접근 불가
- `/districts` 접근 불가
- 자기 데이터 및 허용된 구역 데이터만 접근 가능

### 7.4 pending

- 승인 전 일반 기능 접근 불가
- 승인 대기 화면으로 유도되어야 함

## 8. 실행 순서

### 8.1 범위 선언

이번 QA의 범위를 먼저 고정한다.

예시:

- 권한 회귀 점검
- 운영 시작 전 전체 역할 QA
- 멀티구역 기능 추가 후 역할 검증

### 8.2 공통 사전 확인

역할별 검증에 앞서 아래를 먼저 확인한다.

- 로그인 화면 진입 가능
- 테스트 URL 동작 가능
- 테스트 계정이 존재함
- 필요한 샘플 데이터가 존재함

### 8.3 pending 검증

먼저 `pending` 계정으로 확인한다.

필수 확인 항목:

- 로그인 시 승인 대기 화면으로 이동하는가
- 일반 대시보드 진입이 차단되는가
- 관리자 화면 접근이 차단되는가

### 8.4 member 검증

다음 항목을 확인한다.

- 로그인 가능
- 대시보드 진입 가능
- 성경공부 목록/상세 접근 가능
- 기도제목 등록 및 조회 가능
- 성경읽기 기록 가능
- 일정 및 출석 응답 가능
- `/admin` 접근 차단
- `/districts` 접근 차단
- 다른 구역 데이터 비노출

### 8.5 leader 검증

다음 항목을 확인한다.

- 로그인 가능
- 일반 사용자 기능 사용 가능
- `/admin` 접근 가능
- 구역원 승인/권한 변경 화면 동작 확인
- 성경공부 관리 가능
- 주간 보고 확인 가능
- 알림 발행 가능
- `/districts` 접근 차단
- 타 구역 전체 관리 불가

### 8.6 master 검증

다음 항목을 확인한다.

- 로그인 가능
- `/admin` 접근 가능
- `/districts` 접근 가능
- 구역 생성/수정/비활성화 가능
- 전체 구역 전환 또는 전체 구역 운영 기능 확인
- 멀티구역 데이터 관리 가능

### 8.7 구역 분리 검증

역할별 검증과 별도로 아래를 확인한다.

- 서로 다른 구역 사용자의 데이터가 섞이지 않는가
- 구역 전환 시 현재 구역 기준 데이터만 보이는가
- 성경공부, 기도제목, 일정, 알림, 관리자 데이터가 구역 기준으로 분리되는가

### 8.8 핵심 기능 시나리오 검증

역할별 접근 제어 외에 아래 핵심 시나리오를 확인한다.

#### 인증

- 회원가입
- 승인 후 로그인
- 비밀번호 재설정

#### 성경공부

- 발행 자료 조회
- 답변 저장
- 완료 저장
- 관리자 등록/수정/발행

#### 기도제목

- 등록
- 응답
- 공유
- 중보기도 수 반영

#### 성경읽기

- 일일 기록 저장
- 누적 집계 반영
- 관리자 통계 조회

#### 일정 및 출석

- 일정 등록/수정/삭제
- 출석 응답
- 첨부 업로드

#### 관리자

- 구역원 승인/거절
- 권한 변경
- 주간 보고 조회
- 주간 마감 실행/해제
- CSV 다운로드
- 카카오 공지 생성

#### 알림 및 검색

- 알림 발행
- 읽음 처리
- 삭제
- 전역 검색

### 8.9 실패 항목 분류

실패한 항목은 아래 중 하나로 분류한다.

- `권한 버그`
- `구역 스코프 버그`
- `운영 설정 문제`
- `테스트 데이터 문제`
- `문서 불일치`

그리고 각 실패 항목마다 아래를 남긴다.

- 어떤 역할에서 발생했는가
- 어떤 페이지/기능에서 발생했는가
- 기대 동작은 무엇이었는가
- 실제 결과는 무엇이었는가
- 재현 가능한가

### 8.10 후속 조치 연결

실패 항목 발견 시 아래 중 하나로 즉시 연결한다.

- 버그 수정 작업
- 운영 설정 수정
- 테스트 데이터 보강
- 문서 수정

## 12. 차단 시 처리

다음 상황에서는 중단하지 말고 상태 보고로 마무리한다.

- 테스트 계정이 없음
- 특정 역할 데이터가 준비되지 않음
- 원격 URL 접근이 안 됨
- 첨부 업로드나 이메일 재설정처럼 외부 의존 기능 확인이 불가함

이 경우 아래를 남긴다.

- 확인된 범위
- 확인 실패 원인
- 추가로 필요한 계정/권한/데이터
- 다음 점검 순서

## 14. 메모

- `bethel803`의 역할 QA는 단순 권한 검증이 아니라 운영 준비 검증의 핵심이다.
- 멀티구역 구조가 포함되어 있으므로 구역 분리 검증을 항상 포함해야 한다.
- 관리자 기능 검증은 `leader`와 `master`를 구분해서 수행해야 한다.
